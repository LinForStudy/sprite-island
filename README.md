# 萌灵小岛

《萌灵小岛》是一个原创横屏网页小游戏，核心体验是探索小岛、发现萌灵、捕捉收集、培养成长、穿戴装备、野外对战，并在战斗胜利后尝试收服新的萌灵。

项目不使用宝可梦官方名称、角色、图像、精灵球或相似素材。当前美术以原创 SVG + CSS 动画实现，方便后续继续替换为手绘或 AI 生成的 PNG 资源。

## 当前版本

- 48 只原创动态 SVG 萌灵。
- 12 个探索地点：草丛、池塘、暖石、风车、山洞、云台、花林、珊瑚湾、熔灯台、星磁塔、化石坡、彩云桥。
- 4 个稀有度：常见、稀有、传奇、神话。
- 6 个属性：草、水、火、电、土、风。
- 横屏 16:9 游戏框，面向桌面浏览器和后续 APK 横屏打包。
- 本地浏览器存档，支持导出、导入、备份恢复和清空确认。

## 核心玩法

### 小岛探索

玩家在小岛地图上点击不同地点进行探索。每个地点都有独立的萌灵池，发现和入住进度会显示在地点旁边。

### 捕捉与图鉴

探索时会随机遇见萌灵，可以使用原创道具进行捕捉。图鉴会记录已发现和已入住状态，已入住萌灵可以从图鉴直接进入小屋装备页面。

### 小屋养成

小屋中可以选择已入住萌灵，查看等级、经验、生命、力量、防御、魔法、亲密度、饱腹度和清洁度，并进行喂食、清洁、摸摸和恢复全部生命。

### 野外对战

玩家可以在每个地点发起野外挑战。进入战斗前先选择出战萌灵，野怪等级会根据所选萌灵匹配，避免低等级萌灵被高等级野怪碾压。

战斗为轻量回合制：

- 每只萌灵有生命、力量、防御、魔法和 2 个技能。
- 技能分为物理、魔法和少量治疗类。
- 属性克制关系为：草克土、土克电、电克水、水克火、火克风、风克草。
- 胜利获得经验，升级后属性成长。
- 失败不扣除图鉴和宠物，回小屋恢复即可继续挑战。

### 装备与武器库

战斗胜利有概率掉落装备。装备属于玩家萌灵，野外萌灵默认无装备，用来拉开同名同级战斗时的差异。

装备槽位包括：

- 武器：主要提升力量或魔法。
- 护甲：主要提升生命和防御。
- 护符：提供属性技能、治疗或经验相关加成。
- 鞋子：提供生命、防御或魔法等小幅加成。

同一件装备实例只能给一只萌灵穿戴。如果有多件同款装备，可以分别给不同萌灵穿。

### 战斗后收服

野外挑战胜利后，如果对手还没有入住，会进入战斗后收服流程：

- 先抽取收服球。
- 不同收服球提供不同成功率加成。
- 每次战斗只有一次收服机会。
- 如果对手已经入住，会提示已经收服过，不重复获得同一只萌灵。
- 收服过程包含抽球、抛球、晃动、成功闪光和失败反馈动画。

## 存档说明

游戏数据保存在当前浏览器、当前访问地址的 `localStorage` 中，主 key 为：

```text
sprite-island-save-v1
```

已实现的存档保护：

- 自动保存图鉴、已入住萌灵、等级、经验、生命、养成状态和装备。
- 保存前写入最近备份。
- 主存档损坏时尝试从备份恢复。
- 清空存档前会二次确认，并生成清空前备份。
- 首页提供导出、导入、恢复和清空功能，方便家长管理。

注意：`localhost`、`127.0.0.1`、局域网 IP、不同端口、不同浏览器的 `localStorage` 彼此独立。如果看起来像重置，优先检查是否换了访问地址或浏览器。

- Vue 3

## 本地运行

安装依赖：

```bash
npm.cmd install --cache .\.npm-cache
```

启动开发服务器：

```bash
npm.cmd run dev -- --port 5174
```

打开：

```text
http://127.0.0.1:5174
```

## 构建

```bash
npm.cmd run build
```

构建产物会输出到：

```text
dist
```

## 适配目标

- 桌面端优先适配 1920x1080。
- 小屏横屏保持 16:9 游戏框，内部区域滚动，避免页面级溢出。
- 竖屏显示横屏提示。
- 后续如果打包成 APK，建议保持横屏模式。

## 设计原则

- 面向 1-2 年级孩子，按钮大、文字短、反馈温和。
- 不做联网、不做登录、不做付费抽卡。
- 失败不惩罚收集进度，避免孩子因为输掉战斗卡住。
- 所有角色、美术、道具和命名保持原创。
